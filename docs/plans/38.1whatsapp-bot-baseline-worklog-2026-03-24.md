## Что было обнаружено до исправлений

### 1. Runtime prompt drift
- Код уже содержал новый `OPENAI_PROMPT_VERSION`.
- Но в Railway логах бот продолжал показывать старые runtime значения:
  - `promptVersion = 2026-03-14.1`
  - `policyVersion = 2026-03-14.a`
  - `fastPathVersion = 2026-03-14.a`
- Это означало, что без дополнительной диагностики невозможно надежно понять, какая prompt-ветка реально используется в проде.

### 2. Locale поведение было недостаточно прозрачно
- В логах был только `localeReason`.
- Не хватало детализации:
  - marker scores;
  - inference scores;
  - признака удержания языка через session hold.

### 3. Timeout ошибки выглядели как domain errors
- При AI timeout в логах появлялся `tool_domain_error`.
- Это смешивало:
  - проблемы OpenAI/timeout/chain;
  - реальные domain/tool ошибки.

### 4. Reset detector неверно распознавал некоторые сообщения
- Фраза `cancel my booking` могла попадать в `booking_list`, а не в `cancel_booking`.
- Причина: порядок intent regex в reset detector.

### 5. Non-continuation reset был семантически запутан
- `non_continuation_message` логировался как `hard_reset_to_menu`, хотя фактически сообщение reroute-илось как новый запрос.
