# План 49.1: UAT-чеклист для mobile admin dashboard + WhatsApp quick actions

## Цель
Проверить end-to-end сценарии администрирования записей с телефона/планшета и через WhatsApp-команды администратора.

## Подготовка окружения
1. Деплой `api`, `bot`, `worker`, `web` в одном окружении.
2. Для тестового tenant заданы:
   - `adminNotificationWhatsappE164`;
   - `operatorWhatsappE164`;
   - подключенный WA endpoint в super-admin.
3. В `bot` и `worker` заданы `WEB_URL` (или `APP_URL`) и `INTERNAL_API_SECRET`.
4. В календаре есть минимум 6 записей: pending/confirmed/completed/no_show на сегодня и завтра.

## Блок A: Dashboard mobile/tablet
1. На телефоне открыть `/app/dashboard`:
   - видны блоки `Bookings today` и `Bookings tomorrow`;
   - есть кнопка `Confirm` для pending.
2. Нажать `Confirm` в блоке today:
   - статус меняется в карточке;
   - запись видна как confirmed во вкладке Bookings.
3. Проверить KPI:
   - `No-show today`;
   - `Completed revenue today` (EUR).
4. На планшете проверить split/адаптив:
   - таблицы и кнопки не ломают layout;
   - нет выхода кнопок за границы блоков.

## Блок B: Bookings actions
1. Для pending:
   - `Confirm` работает;
   - `Cancel` требует reason;
   - `Reject` требует reason.
2. Для confirmed:
   - `Complete` работает с пустой суммой;
   - `Complete` с суммой > 0 сохраняется;
   - `No-show` переводит запись в `no_show`.
3. Попытка `Complete` с `0`/отрицательной суммой:
   - возвращается валидационная ошибка.
4. Фильтр `No-show`:
   - показывает записи со статусом `no_show`.

## Блок C: WhatsApp quick actions (admin)
1. С номера администратора отправить `today`:
   - приходит список записей на сегодня;
   - ниже приходят кнопки quick actions.
2. Нажать `Tomorrow`:
   - приходит список на завтра.
3. Нажать `Next`:
   - приходит ближайшие 3 записи.
4. Если в ответе есть pending:
   - бот дает CTA `Confirm/Reject` для ближайшей pending;
   - `Confirm` меняет статус в админке;
   - `Reject` запрашивает причину и отправляет ее клиенту.
5. Нажать `Open web`:
   - бот присылает ссылку `/app/bookings`.

## Блок D: Worker admin notification
1. Создать новую booking клиентом.
2. На admin WhatsApp приходит `booking_created_admin`:
   - есть CTA `Confirm/Reject`;
   - в session-mode есть `Open web`.
3. Нажать `Confirm` из уведомления:
   - статус записи меняется на confirmed;
   - клиент получает confirmation.

## Блок E: Регрессии
1. Обычный клиентский flow (create/reschedule/cancel) не сломан.
2. Reminder flow 24h/2h отправляется как раньше.
3. Логи не содержат массовых ошибок `admin_digest_failed`/`booking_admin_action_failed`.

## Критерий приемки
Все пункты A-E пройдены без ручного вмешательства в БД и без hotfix в процессе теста.
